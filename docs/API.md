## viewer_types

Shared dataclasses and structural protocols used across the viewer. Nothing here does I/O — these are the type contracts that the rest of the modules agree on.

### `MeshGeometry`

Mesh geometry for deck.gl rendering. Produced by `build_mesh_geometry()`, consumed by all layer builders and analysis functions. Positions and indices are base64-encoded for efficient WebSocket transport; `METER_OFFSETS` layers expect coordinates relative to `(lon_off, lat_off)`.

**Fields**:
- `npoin: int` — number of 2D mesh nodes
- `positions: dict[str, Any]` — binary-encoded vertex positions for deck.gl
- `indices: dict[str, Any]` — binary-encoded triangle indices for deck.gl
- `x_off: float` — mesh center X in native CRS (meters)
- `y_off: float` — mesh center Y in native CRS (meters)
- `lon_off: float` — WGS84 longitude of mesh center (0.0 if no CRS)
- `lat_off: float` — WGS84 latitude of mesh center (0.0 if no CRS)
- `crs: CRS | None` — coordinate reference system
- `extent_m: float` — mesh bounding-box extent in meters
- `zoom: float` — suggested deck.gl zoom level

Frozen dataclass.

### `TelemacFileProtocol`

Structural type for `TelemacFile`-like objects. Used everywhere in the codebase as the type hint instead of importing the concrete `TelemacFile` class, so tests can pass in lightweight stubs.

**Fields / attributes**:
- `meshx: np.ndarray` — X coordinates of mesh nodes
- `meshy: np.ndarray` — Y coordinates of mesh nodes
- `npoin2: int` — number of 2D nodes
- `nelem2: int` — number of 2D elements
- `ikle2: np.ndarray` — connectivity table, shape `(nelem2, 3)`
- `varnames: list[str]` — variable names in the file
- `times: Sequence[float]` — timestep values (seconds)
- `nplan: int` — number of vertical planes (1 for 2D)

**Methods**:
- `get_data_value(varname: str, tidx: int) -> np.ndarray`
- `get_z_name() -> str`
- `get_timeseries_on_points(varname: str, points: list) -> list`
- `get_data_on_points(varname: str, tidx: int, points: list) -> list`
- `get_data_on_polyline(varname: str, tidx: int, polyline: list) -> tuple`
- `close() -> None`

## geometry

Builds the `MeshGeometry` object that every layer and analysis module consumes. Centers the mesh on its bounding-box midpoint and, when a CRS is supplied, projects that center to WGS84 for deck.gl basemap alignment.

### `build_mesh_geometry(tf: TelemacFileProtocol, crs: CRSType | None = None, z_values: np.ndarray | None = None, z_scale: float = 1, origin_offset: tuple[float, float] = (0, 0)) -> MeshGeometry`

Build mesh geometry for `SimpleMeshLayer`.

Transforms TELEMAC mesh coordinates (arbitrary metric CRS) into centered-meter coordinates by subtracting the bounding-box center (`x_off`, `y_off`). All layer rendering uses these centered coordinates via deck.gl's `METER_OFFSETS` coordinate system.

When a CRS is provided, the mesh center is also converted to WGS84 (`lon_off`, `lat_off`) for basemap alignment.

If `z_values` and `z_scale` are provided, `positions[:, 2]` is set to `z_values * z_scale` for 3D visualization.

Returns a `MeshGeometry` whose `positions` and `indices` are binary-encoded dicts (base64 transport for efficient WebSocket transfer), plus `x_off`/`y_off` (mesh center for round-trip), `lon_off`/`lat_off` (WGS84 origin for deck.gl), `extent_m`, `zoom`, and `crs`. Also honours SELAFIN `I_ORIG`/`J_ORIG` offsets found in `tf.iparam[2:4]`.

## constants

Environment bootstrap, example registry, palettes, and small formatting helpers. Importing this module also sets `HOMETEL`, `SYSTELCFG`, and `USETELCFG` environment variables if they are not already defined, and inserts the TELEMAC Python scripts directory on `sys.path`.

Module-level constants:

- `EXAMPLES: dict[str, str]` — map of example name to file path, filtered to files that actually exist on disk.
- `EXAMPLE_GROUPS: dict[str, dict[str, str]]` — examples grouped by TELEMAC module (TELEMAC-2D, TELEMAC-3D, GAIA, TOMAWAC, ARTEMIS, KHIONE, Case Studies).
- `EXAMPLE_CHOICES: dict[str, dict[str, str]]` — same structure as `EXAMPLE_GROUPS`, restricted to existing files, formatted for a grouped Shiny dropdown.
- `PALETTES: dict[str, list]` — ten named color palettes re-exported from `shiny-deckgl` (`Viridis`, `Plasma`, `Ocean`, `Thermal`, `Chlorophyll`, `Blues`, `Greens`, `Reds`, `Yellow-Red`, `Blue-White`).
- `PALETTE_DIVERGING: list` — blue-white-red diverging palette used for difference mode.
- `MAP_BG_DARK`, `MAP_BG_LIGHT: str` — MapLibre data-URI styles for the default dark/light flat backdrops.
- `BASEMAP_STYLES: dict[str, str]` — selector entries for the sidebar (dark, light, osm, satellite).
- `MAP_TOOLTIP: dict` — deck.gl tooltip styling used by the map widget.

### `solid_bg_style(hex_color: str) -> str`

Return a MapLibre data-URI style that renders a flat `hex_color` background. Used instead of a tiled basemap when we only want the deck.gl layers to sit on a solid backdrop (the TELEMAC app's default look).

### `cached_palette_arr(palette_id: str, reverse: bool = False) -> np.ndarray`

LRU-cached (maxsize=16) 256-entry RGBA `uint8` array for per-vertex coloring. Pass `palette_id="_diverging"` for the blue-white-red difference palette. Unknown palette IDs fall back to `Viridis` with a warning. When `reverse=True`, the array is returned with entries reversed.

### `cached_gradient_colors(palette_id: str, reverse: bool = False) -> list[list[int]]`

LRU-cached (maxsize=16) list of 8 evenly-sampled color stops from the full 256-color palette, used to render legend gradients in the UI. Handles `"_diverging"` the same as `cached_palette_arr`.

### `format_time(seconds: float) -> str`

Human-readable time formatting. Under 60 seconds: `"12.3 s"`. Under 60 minutes: `"15 min 30 s"`. Otherwise: `"3 h 15 min"`.

## crs

Single source of truth for all CRS operations: EPSG lookup, native↔WGS84 transforms, `.cas` auto-detection, click conversion.

### `CRS`

Frozen dataclass wrapping a pyproj `Transformer` pair.

**Fields**:
- `epsg: int` — EPSG code (e.g. 2154 for Lambert-93)
- `name: str` — human-readable name (from pyproj)
- `transformer: Transformer` — native → WGS84
- `inv_transformer: Transformer` — WGS84 → native

`__post_init__` raises `ValueError` if either transformer is `None`.

### `crs_from_epsg(code: int) -> CRS`

Create `CRS` from EPSG code. Raises `pyproj.exceptions.CRSError` for invalid codes.

### `native_to_wgs84(x: float, y: float, crs: CRS) -> tuple[float, float]`

Transform native CRS coordinates to WGS84 `(lon, lat)`.

### `wgs84_to_native(lon: float, lat: float, crs: CRS) -> tuple[float, float]`

Transform WGS84 `(lon, lat)` to native CRS coordinates.

### `detect_crs_from_cas(cas_path: str) -> CRS | None`

Parse TELEMAC `.cas` file for `GEOGRAPHIC SYSTEM` (or `SYSTEME GEOGRAPHIQUE`) and `ZONE NUMBER IN GEOGRAPHIC SYSTEM` keywords. Returns `CRS` if detected, `None` otherwise. Handles TELEMAC's comment syntax (`/`). Maps geosyst codes to EPSG as follows: 1→WGS84, 2→UTM North, 3→UTM South, 4→Lambert (zone-dependent), 5→World Mercator 3395.

### `guess_crs_from_coords(x: ndarray, y: ndarray) -> CRS | None`

Heuristic CRS guess from coordinate ranges. Currently only detects LKS94 (Lithuania, EPSG:3346). Returns `None` for anything ambiguous — WGS84 in particular cannot be detected from coordinates alone because small lab models (0–100 m) are indistinguishable from geographic degrees.

### `click_to_native(lon: float, lat: float, geom: Any) -> tuple[float, float]`

Convert deck.gl click coordinates to native CRS meters. When `geom.crs` is set, deck.gl reports real lon/lat and pyproj inverse is used. When `geom.crs` is `None`, falls back to the `_M2D` approximation that reverses the meters-to-degrees scaling applied in `build_mesh_geometry`.

### `meters_to_wgs84(x_m: float, y_m: float, geom: Any) -> tuple[float, float] | None`

Convert native CRS meters to WGS84 for display. Returns `None` if `geom.crs` is `None`.

## telemac_defaults

TELEMAC variable semantics: palette defaults, module detection, velocity-pair resolution. Pure Python, no numpy — just string matching against TELEMAC's canonical variable-name conventions in English and French.

Module-level tables:

- `VARIABLE_PALETTE_MAP: list[tuple[str, str]]` — ordered `(substring, palette)` pairs matched case-insensitively; longer/more specific patterns come first.
- `BIPOLAR_VARIABLES: set[str]` — variable names that should use a symmetric color range (`EVOLUTION`, bed-evolution variants).
- `MODULE_SIGNATURES: dict[str, list[str]]` — per-module variable signatures used to identify the producing TELEMAC module from a result file's variable list.
- `VELOCITY_PAIRS: list[tuple[str, str]]` — the ordered list of `(U, V)` variable-name pairs we recognise (`VELOCITY U/V`, `UX/UY`, `VX/VY`, `QSBLX/QSBLY`).

### `suggest_palette(varname: str) -> str | None`

Return suggested palette name for a variable, or `None` for default.

### `is_bipolar(varname: str) -> bool`

Check if a variable is bipolar (erosion/deposition, positive/negative).

### `detect_module_from_vars(varnames: list[str]) -> str`

Detect TELEMAC module from variable names. Returns `"TELEMAC-2D"` as fallback when no module signature matches.

### `find_velocity_pair(varnames: list[str]) -> tuple[str, str] | None`

Find the first available velocity U/V pair from variable names. Returns `(u_name, v_name)` using the exact names from the file, or `None` if no velocity pair is found.

## layers

deck.gl layer builders. All builders return dict-based layer specifications for shiny-deckgl's `MapWidget`, using `COORDINATE_SYSTEM.METER_OFFSETS` so layer `x/y` are meters relative to a WGS84 origin set per-layer via `coordinateOrigin`.

### `build_mesh_layer(geom: MeshGeometry, values: np.ndarray, palette_id: str, filter_range: tuple[float, float] | None = None, color_range_override: tuple[float, float] | None = None, log_scale: bool = False, reverse_palette: bool = False, origin: list[float] | None = None) -> tuple[dict, float, float, bool]`

Build `SimpleMeshLayer` with per-vertex coloring and optional value filter. Returns `(layer_dict, vmin, vmax, log_applied)`. `log_applied` indicates whether log scaling was actually applied (requires strictly positive `vmin` and a non-degenerate range).

### `build_mesh_color_patch(geom: MeshGeometry, values: np.ndarray, palette_id: str, filter_range: tuple[float, float] | None = None, color_range_override: tuple[float, float] | None = None, log_scale: bool = False, reverse_palette: bool = False) -> tuple[dict, float, float, bool]`

Build a partial-update patch for the mesh layer (colors only). Returns a dict suitable for `map_widget.partial_update([patch])`. The JS-side layer cache preserves positions, indices, origin, and coordinate system — only the `_meshColors` binary attribute is resent. For a 100k-node float32 mesh this cuts roughly 60–70% of the per-tick payload when the user scrubs through time or switches palette.

### `build_velocity_patch(tf: TelemacFileProtocol, tidx: int, geom: MeshGeometry, origin: list[float] | None = None) -> dict | None`

Partial-update patch for the velocity arrow layer. Today this is a pass-through to `build_velocity_layer` — velocity arrows are cheap relative to mesh colors, so we resend the full layer dict. The wrapper exists so the fast path in `update_map` can call `build_*_patch` uniformly; if we later introduce sparse attribute updates for arrows, only this function changes.

### `build_contour_patch(tf: TelemacFileProtocol, values: np.ndarray, geom: MeshGeometry, n_contours: int = 6, layer_id: str = "contours", contour_color: list[int] | None = None, origin: list[float] | None = None) -> dict | None`

Partial-update patch for a contour layer. Today this is a pass-through to `build_contour_layer_fn` — contours are recomputed from scratch when the field changes because the marching-triangles pass produces entirely new crossing points per tick. A true sparse update would require segment-level diffing that isn't worth the complexity yet; the seam exists so the map dispatcher can call `build_*_patch` uniformly.

### `build_velocity_layer(tf: TelemacFileProtocol, tidx: int, geom: MeshGeometry, origin: list[float] | None = None) -> dict | None`

Build velocity arrow layer from U/V components. Auto-detects the velocity variable pair via `find_velocity_pair`. Subsamples to roughly 1500 arrows; scales arrow length so the longest is about 1/30 of the mesh extent. Returns `None` if no velocity pair is available or the field is effectively zero.

### `build_contour_layer_fn(tf: TelemacFileProtocol, values: np.ndarray, geom: MeshGeometry, n_contours: int = 6, layer_id: str = "contours", contour_color: list[int] | None = None, origin: list[float] | None = None) -> dict | None`

Build FEM-exact contour lines using marching triangles. Walks each triangle to find edges where the field value crosses contour thresholds, interpolates the exact crossing point, and builds line segments. Much more accurate than grid-based KDE contouring for unstructured meshes.

### `build_marker_layer(x_m: float, y_m: float, layer_id: str = "marker", origin: list[float] | None = None) -> dict`

Build a single-point scatterplot layer to mark clicked location.

### `build_cross_section_layer(points_m: list[list[float]], origin: list[float] | None = None) -> dict`

Build a `PathLayer` showing the cross-section polyline on the map. `points_m` is a list of `[x, y]` in meters relative to mesh center.

### `build_particle_layer(paths: list[list[list[float]]], current_time: float, trail_length: float, origin: list[float] | None = None) -> dict`

Build a `TripsLayer` for particle trace animation. `paths` is a list of particle trajectories, each a list of `[x, y, timestamp]` with coordinates in meters relative to mesh center.

### `build_wireframe_layer(tf: TelemacFileProtocol, geom: MeshGeometry, origin: list[float] | None = None) -> dict`

Build mesh wireframe as line segments (triangle edges). Subsamples to a 50 000-edge cap to keep the draw call tractable on big meshes.

### `build_extrema_markers(extrema: dict[str, tuple], x_off: float, y_off: float, origin: list[float] | None = None) -> list[dict]`

Build scatterplot markers for min/max value locations. Expects the `extrema` dict returned by `find_extrema` (keys `"min"`, `"max"`, each a `(node_idx, x_m, y_m, value)` tuple).

### `build_measurement_layer(points_m: list[list[float]], origin: list[float] | None = None) -> list[dict]`

Build a line + endpoint markers for distance measurement. Returns two layers (line + scatter) when two points are provided; one (endpoint scatter only) when a single point is given.

### `build_boundary_layer(tf: TelemacFileProtocol, geom: MeshGeometry, boundary_nodes: list[int], bc_types: dict[int, int] | None = None, boundary_edges: tuple | None = None, origin: list[float] | None = None) -> list[dict]`

Build boundary edge lines color-coded by hydrodynamic type. Returns a list of layers: one line layer per boundary type plus a scatterplot layer for prescribed-value nodes. LIHBOR codes: 2=wall (gray), 4=free/Neumann (green), 5=prescribed (blue). `bc_types` is the node→LIHBOR mapping returned by `read_cli_file`; `boundary_edges` is the tuple returned by `find_boundary_edges` (computed on demand if not supplied).

### `build_polygon_layer(polygon_coords: list[list[float]], origin: list[float] | None = None) -> dict`

Build a `GeoJsonLayer` outlining the user-drawn polygon. `polygon_coords` is a list of `[x_m, y_m]` points in mesh-offset coordinates; the ring is closed automatically.

## analysis

Spatial and temporal analysis on TELEMAC result files. Function boundaries match the UI panels in `app.py`: derived variables, point/polyline extraction, discharge, mesh diagnostics, flood mapping, temporal stats, particle tracing, and file I/O helpers.

Module-level tables:

- `DERIVED_VARIABLES: dict` — recipe table for on-the-fly derived variables (`VELOCITY MAGNITUDE`, `FROUDE NUMBER`, `VORTICITY`); each entry lists required variable names and the compute function (or `"vorticity"` sentinel for mesh-gradient-based derivation).

### `get_available_derived(tf: TelemacFileProtocol) -> list[str]`

Return list of derived variable names available for this file. A derived variable is "available" when every variable listed in its `requires` list appears in `tf.varnames`.

### `compute_derived(tf: TelemacFileProtocol, varname: str, tidx: int) -> np.ndarray`

Compute a derived variable. Returns numpy array. Dispatches to the lambda in `DERIVED_VARIABLES`, or to the mesh-gradient vorticity routine for `"VORTICITY"`.

### `get_var_values(tf: TelemacFileProtocol, varname: str, tidx: int) -> np.ndarray`

Get variable values, handling both file and derived variables. Use this instead of `tf.get_data_value()` when the variable name may be a derived quantity (e.g. `VELOCITY MAGNITUDE`, `FROUDE NUMBER`, `VORTICITY`).

### `export_timeseries_csv(times: np.ndarray, values: np.ndarray, varname: str) -> str`

Format time series data as CSV string with a `Time (s), <varname>` header.

### `export_crosssection_csv(abscissa: np.ndarray, values: np.ndarray, varname: str) -> str`

Format cross-section data as CSV string with a `Distance (m), <varname>` header.

### `compute_discharge(tf: TelemacFileProtocol, tidx: int, polyline_m: list[list[float]]) -> dict[str, Any]`

Compute discharge (Q) through a cross-section polyline. `Q = integral of (velocity · normal) * depth` along the section. Returns dict with `total_q` (m³/s), per-segment `segments` breakdown, and `skipped` count. Returns an error dict with `total_q=None` and an `error` message when VELOCITY U/V or WATER DEPTH is missing.

### `mesh_identity_hash(tf: TelemacFileProtocol) -> str`

Short SHA1 digest (12 hex chars) identifying mesh geometry `(x, y, ikle)`. Two files with bit-identical 2D coordinates and connectivity produce the same hash; any difference — including float noise from re-export — yields a different hash. This is intentional: compare-overlay must reject any non-identical mesh to avoid rendering file B's values on file A's geometry.

### `compute_courant_number(tf: TelemacFileProtocol, tidx: int) -> np.ndarray | None`

Compute Courant number (CFL) per vertex: `CFL = V * dt / dx`. `dx` is estimated as `sqrt(average element area around each vertex)`. Note: `dt` is the output file interval between saved timesteps, not the solver's internal computational timestep. The true CFL governing numerical stability may be much smaller. This metric is useful for comparing relative flow intensity across the domain. Returns `None` when no velocity pair is available.

### `compute_element_area(tf: TelemacFileProtocol) -> np.ndarray`

Compute per-vertex element area (average of adjacent element areas).

### `compute_mesh_integral(tf: TelemacFileProtocol, values: np.ndarray, threshold: float | None = None) -> dict[str, float]`

Compute area-weighted integral and statistics over the mesh. If `threshold` is set, only elements where all vertices exceed it are included in `wetted_area`. Returns dict with `total_area`, `integral`, `mean`, `wetted_area`, `wetted_fraction`.

### `evaluate_expression(tf: TelemacFileProtocol, tidx: int, expression: str) -> np.ndarray`

Evaluate a user-defined math expression safely using AST parsing (no `eval`). Variables are referenced by name with spaces replaced by underscores (e.g., `"VELOCITY_U**2 + VELOCITY_V**2"`). Available functions: `sqrt`, `abs`, `log`, `log10`, `exp`, `sin`, `cos`, `tan`, `maximum`, `minimum`, `where`, `pi`. Supports comparisons and conditional (`x if cond else y`) expressions. Raises `ValueError` on unsupported constructs.

### `compute_slope(tf: TelemacFileProtocol, values: np.ndarray) -> np.ndarray`

Compute slope magnitude (gradient) of a scalar field on the mesh. Uses per-element gradient averaged to vertices. Returns per-vertex slope array (float32).

### `export_all_variables_csv(tf: TelemacFileProtocol, tidx: int, x_m: float, y_m: float) -> str`

Export all variable values at a point for a given timestep as CSV string. Samples at the nearest mesh node.

### `find_boundary_nodes(tf: TelemacFileProtocol) -> list[int]`

Find boundary nodes of the 2D mesh. Boundary edges appear in only one triangle. Returns sorted list of unique node indices.

### `compute_unique_edges(ikle: np.ndarray) -> tuple[np.ndarray, np.ndarray, np.ndarray]`

Extract unique edges from triangle connectivity. Returns `(edge_keys, nodes_a, nodes_b)` where `nodes_a[i]`–`nodes_b[i]` form the i-th unique edge.

### `find_boundary_edges(tf: TelemacFileProtocol) -> tuple[np.ndarray, np.ndarray, np.ndarray]`

Find boundary edges of the 2D mesh. Returns `(boundary_keys, nodes_a, nodes_b)` where `nodes_a[i]`–`nodes_b[i]` form the i-th boundary edge.

### `read_cli_file(cli_path: str) -> dict[int, int] | None`

Read TELEMAC `.cli` boundary condition file. Returns dict mapping 1-based `node_number` to LIHBOR code: 2 = solid wall, 4 = free (Neumann), 5 = prescribed H or Q. Returns `None` if file cannot be read or parsed.

### `extract_layer_2d(values_3d: np.ndarray, npoin2: int, layer_k: int) -> np.ndarray`

Extract a single horizontal layer from 3D variable data. For TELEMAC-3D files with `nplan` vertical planes, variable arrays have length `npoin2 * nplan`. This extracts plane k as a 2D array of length `npoin2`. `layer_k=0` is the bottom; `layer_k=nplan-1` is the surface.

### `polygon_zonal_stats(tf: TelemacFileProtocol, values: np.ndarray, polygon_m: list[list[float]], flood_threshold: float = 0.01, var_name: str = "") -> dict[str, float] | None`

Compute statistics of a variable within a drawn polygon. `polygon_m` is a list of `[x_m, y_m]` vertices in mesh meters. `flood_threshold` is the depth threshold for flooded fraction. `var_name` is the variable name; flooded metrics are only computed for depth-like vars (names containing `DEPTH` or `HAUTEUR`). Returns dict with `area`, `mean` (element-area weighted), `min`, `max`, `count`, `flooded_area`, `flooded_fraction`. Returns `None` when the polygon contains no mesh nodes (empty intersection), so the caller can distinguish that from a polygon over genuine zeros.

### `nearest_node(tf: TelemacFileProtocol, x_m: float, y_m: float) -> tuple[int, float, float]`

Find the nearest 2D mesh node to a point in mesh meters. Returns `(node_index, node_x, node_y)`. Raises `ValueError` for an empty mesh.

### `time_series_at_point(tf: TelemacFileProtocol, varname: str, x_m: float, y_m: float) -> tuple[np.ndarray, np.ndarray]`

Extract time series for a variable at a mesh point. Returns `(times, values)` arrays. Supports derived variables via barycentric interpolation within the enclosing triangle, matching the semantics of raw `tf.get_timeseries_on_points`. Falls back to nearest node if the point lies outside the mesh.

### `cross_section_profile(tf: TelemacFileProtocol, varname: str, record: int, polyline_m: list[list[float]]) -> tuple[np.ndarray, np.ndarray]`

Extract variable values along a polyline. `polyline_m` is a list of `[x, y]` in mesh meters. Returns `(curvilinear_abscissa, values)`. Supports derived variables by interpolating with `scipy.interpolate.LinearNDInterpolator` onto the polyline points produced by TELEMAC.

### `compute_particle_paths(tf: TelemacFileProtocol, seed_points: list[list[float]], x_off: float, y_off: float) -> list[list[list[float]]]`

Compute Lagrangian particle trajectories from velocity field. Uses batched interpolation: fetches velocity field once per timestep and interpolates all active particles simultaneously. `seed_points` is a list of `[x_m, y_m]` in mesh meters. Returns list of paths, each path being `[[x_centered, y_centered, time], ...]`. Particles that leave the mesh or hit a NaN velocity are deactivated.

### `generate_seed_grid(tf: TelemacFileProtocol, n_target: int = 500) -> list[list[float]]`

Generate a regular grid of seed points within the mesh bounding box. Returns list of `[x_m, y_m]` in mesh meters, filtered to points inside the mesh via the matplotlib trifinder. Grid density adapts to mesh aspect ratio; total point count is capped at `4 * n_target` to guard against degenerate meshes.

### `distribute_seeds_along_line(polyline_m: list[list[float]], n_seeds: int = 100) -> list[list[float]]`

Distribute seed points evenly along a polyline. `polyline_m` is a list of `[x, y]` in mesh meters. Returns list of `[x, y]` seed points.

### `compute_mesh_quality(tf: TelemacFileProtocol) -> np.ndarray`

Compute per-vertex mesh quality (0 = worst, 1 = best). Uses element aspect ratio: ratio of inscribed to circumscribed circle radii, normalized so an equilateral triangle is 1.0. Averaged to vertices.

### `vertical_profile_at_point(tf: TelemacFileProtocol, varname: str, tidx: int, x_m: float, y_m: float) -> tuple[np.ndarray, np.ndarray, str]`

Extract vertical profile at a point for 3D files. Returns `(elevations, values, elevation_label)` arrays for each vertical plane. `elevation_label` falls back to `"Layer index (elevation unavailable)"` when the Z variable is absent.

### `find_extrema(tf: TelemacFileProtocol, values: np.ndarray) -> dict[str, tuple]`

Find locations of min and max values on the 2D mesh. Returns dict with `min`/`max` keys, each containing `(node_idx, x_m, y_m, value)`. For an empty or all-NaN field, returns `(-1, 0.0, 0.0, nan)` for both keys.

### `compute_all_temporal_stats(tf: TelemacFileProtocol, varname: str, threshold: float = 0.01) -> dict[str, np.ndarray]`

Single-pass computation of all temporal statistics. Returns dict with keys `max_values`, `min_values`, `mean_values`, `envelope`, `arrival`, `duration`. Runs in one pass over the timesteps — use this instead of combining `compute_flood_envelope`/`compute_flood_arrival`/`compute_flood_duration` when you need more than one of them.

### `compute_temporal_stats(tf: TelemacFileProtocol, varname: str) -> dict[str, np.ndarray] | None`

Compute min, max, mean across all timesteps for a variable. Returns dict with `min`, `max`, `mean` arrays (per-node). Delegates to `get_var_values`, so derived variables are supported.

### `compute_flood_envelope(tf, varname, threshold=0.01) -> np.ndarray`

Compute maximum value over all timesteps per node. Returns per-node array of the peak value across the entire simulation. Nodes where the peak is below `threshold` are set to 0.

### `compute_flood_arrival(tf, varname="WATER DEPTH", threshold=0.01) -> np.ndarray`

Compute flood arrival time per node. Returns per-node array of the simulation time (seconds) when the value first exceeds `threshold`. Nodes never exceeding `threshold` get `NaN`.

### `compute_flood_duration(tf, varname="WATER DEPTH", threshold=0.01) -> np.ndarray`

Compute total flood duration per node. Returns per-node array of total time (seconds) spent above `threshold`. Accumulates forward intervals for each wet timestep except the last.

### `compute_difference(tf: TelemacFileProtocol, varname: str, tidx: int, ref_tidx: int) -> np.ndarray`

Compute difference between current and reference timestep. Uses `get_var_values`, so derived variables are supported.

### `find_cas_files(example_path: str) -> dict[str, str]`

Find `.cas` steering files in the same directory as an example `.slf` file. Returns dict mapping basename to full path.

### `detect_module_from_path(cas_path: str) -> str`

Detect which TELEMAC module a `.cas` file belongs to based on its directory (looks for `examples/<module>/...`). Falls back to filename-prefix heuristics (`t3d` → telemac3d, `art` → artemis, `tom`/`fom` → tomawac); otherwise returns `"telemac2d"`.

## validation

Observation-data I/O and validation metrics. Small module, intentionally decoupled from `analysis` so the UI's "validate" tab can import metrics without pulling in the full analysis toolbox.

### `parse_observation_csv(file_path: str) -> tuple[ndarray, ndarray, str]`

Read a 2-column CSV file with time (seconds) and observed values. The first row must be a header. The second column name becomes the returned `varname`.

- `file_path` — path to the CSV file.
- Returns `(times, values, varname)` where `times` and `values` are 1-D float64 arrays and `varname` is the second column header.

Raises `ValueError` for empty files, files with fewer than 2 columns, or files with no finite data rows.

### `compute_rmse(model: ndarray, observed: ndarray) -> float`

Root Mean Square Error between model and observed arrays.

- `model`, `observed` — arrays of the same shape.
- Returns the RMSE as a float. Raises `ValueError` on shape mismatch.

### `compute_nse(model: ndarray, observed: ndarray) -> float`

Nash-Sutcliffe Efficiency: `NSE = 1 - sum((model - observed)^2) / sum((observed - mean(observed))^2)`. A value of 1.0 indicates a perfect match; can be negative for very poor models. When the observed variance is zero, returns 1.0 if residuals are zero, `-inf` otherwise. Raises `ValueError` on shape mismatch.

### `compute_volume_timeseries(tf, compute_integral_fn) -> tuple[ndarray, ndarray]`

Compute total water volume at each timestep. Uses `compute_integral_fn(tf, values, threshold)` for area-weighted integration. Depth source, in priority order:

1. A direct depth variable (`WATER DEPTH` / `HAUTEUR D'EAU` / `WATER DEPTH M`).
2. Fallback: `FREE SURFACE - BOTTOM` per timestep (logged once).
3. Last resort: first variable in the file (preserves old behaviour).

`FREE SURFACE` alone is NOT water depth (it is bottom + depth), so it must not be integrated directly as volume. Returns `(times, volumes)`.

### `parse_liq_file(liq_path) -> dict | None`

Parse TELEMAC `.liq` liquid boundary file. Returns dict mapping column name (e.g. `'Q(1)'`, `'SL(2)'`) to `{"times": ndarray, "values": ndarray, "unit": str}`. Returns `None` if the file cannot be parsed. Rows with non-numeric or non-finite values are skipped with a warning.
